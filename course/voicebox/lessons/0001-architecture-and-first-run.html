Lesson 1 · Milestone 1

The sidecar shape of voicebox

You already know React. Good — because voicebox's user interface is a React app. The thing that makes voicebox voicebox is everything around that UI: a desktop shell that bundles a Python server, and that server runs the AI voice models. This lesson fixes the single most important idea in the course — the sidecar architecture — and gets the app running on your Mac so every later lesson has a real thing to point at.

The one diagram to memorize

Three processes, one app. The front-end you'd write today; the other two are what you're here to learn.

┌──────────────────────────────────────────────────────────┐ │ TAURI DESKTOP APP │ │ │ │ ┌──────────────┐ ┌───────────────────┐ │ │ │ React UI │ Tauri IPC │ Rust (src-tauri) │ │ │ │ app/ + web/ │◄────────────────►│ window · tray · │ │ │ │ (you know │ │ hotkey · spawns ──┼─┐ │ │ this part) │ │ sidecar │ │ │ └──────────────┘ └───────────────────┘ │ │ └──────────────────────────────────────────────────────────┘ │ │ launch + manage ▼ ┌──────────────────────────────────────────────────────────┐ │ PYTHON SIDECAR (FastAPI · uvicorn · :17493) │ │ backend/ routes · services · backends · database │ │ │ │ │ ▼ loads + runs │ │ torch · MLX ──► 7 TTS engines · STT · cloning · LLM │ └──────────────────────────────────────────────────────────┘ (bundled as a binary via PyInstaller — M9)

Read it as a request flowing right-to-left for output: UI asks the Rust shell → Rust calls the Python sidecar over HTTP on port 17493 → Python runs the model → audio streams back to the UI. Input (dictation) is the reverse. Two languages (Rust, Python) plus the React you know — glued by a localhost HTTP boundary.

The monorepo map

voicebox is a Bun workspace. Match each folder to the diagram:

  • app/ + web/ — the React + Vite UI (your beachhead).
  • tauri/ (src/ = Tauri-bound front-end, src-tauri/ = Rust shell) — the desktop wrapper.
  • backend/ — the Python FastAPI sidecar (models, routes, services). This is where the AI lives.
  • landing/ — marketing site. Out of scope for us.
  • scripts/ + justfile — build orchestration (sidecar build, dev, release).

Run it

You already have the repo at /Users/hongji/voicebox. The project ships a justfile as its canonical orchestrator.

cd /Users/hongji/voicebox
just setup      # python venv + js deps (+ sidecar prereqs)
just dev        # ensure venv + sidecar, then `tauri dev`

If you don't have Tauri's prerequisites or just/bun yet, that's expected — getting them installed is part of M1's setup, and exactly the kind of thing to ask me about. As a lighter first look (UI only, no shell/models):

bun run dev:web   # just the React UI, in a browser

Your tangible win

just dev opens the voicebox window. You see the React UI you could have written, hosted by a Rust shell you couldn't — yet. Type some text, pick a voice, hear it speak. Then close your laptop and draw the sidecar diagram above from memory. If you can do both, M1 is done: you know what voicebox is and where each later lesson lives inside it.
